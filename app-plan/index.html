Overview

Purpose

StitchCraft is an online store for a small family business that sells nursing scrubs and custom embroidered jackets. Right now my parents take orders mostly by word of mouth, so the goal of the site is to give them an actual storefront. Customers can browse what's available, filter by what they're looking for, and add items to a cart instead of having to message someone to place an order.

Audience

The main audience is nurses and other healthcare workers who need scrubs that are comfortable and hold up to long shifts. The second group is people who want a custom jacket, whether that's for a clinic team, a club, or just a gift. A lot of these people are going to be on their phone when they find the site, so it has to work well on mobile and not be a pain to get through.

Dynamic elements

Most of the interactive stuff on the site runs on JavaScript. Instead of hard coding every product into the HTML, I'm keeping all the products in an array inside a data.js file and looping over that array to build the product grid. That way when my parents add a new product later, they only have to update one file.

The filter buttons (All, Scrubs, Jackets) and a sort by price option just rebuild the grid from that same array whenever you click them. Each product has an "Add to cart" button that pushes the item into a cart array and updates the item count and total up in the header. I'm saving the cart to localStorage so it doesn't get wiped if you leave and come back. There's also a hamburger menu for smaller screens, and the contact form checks that the fields aren't empty before it shows a confirmation message.

Typography

Headings use Fraunces, which is a serif that fits the handmade feel of custom clothing. Body text uses Inter since it stays easy to read at small sizes, which matters because most people will be on their phone.

Pseudocode — Computational Thinking

Loading and displaying products

IMPORT products array from data.js
SELECT the grid container element from the DOM

FUNCTION renderProducts(list):
  SET grid.innerHTML to empty string
  FOR EACH product IN list:
    BUILD a card with image, name, price, and an "Add to cart" button
    APPEND the card to the grid
  END FOR
END FUNCTION

CALL renderProducts(products) once on page load

Filtering and sorting

FOR EACH filter button:
  ON click:
    READ the category from the button's data attribute
    IF category is "all":
      SET filtered = products
    ELSE:
      SET filtered = products where product.category equals category
    IF sort is active: SORT filtered by price
    CALL renderProducts(filtered)
  END ON click
END FOR

Cart logic

LOAD cart array from localStorage, OR start with an empty array

FUNCTION addToCart(productId):
  FIND product by id
  IF product already in cart: increase its quantity by 1
  ELSE: push product into cart with quantity 1
  SAVE cart to localStorage
  UPDATE cart count and total in the header
END FUNCTION

Contact form validation

ON form submit:
  PREVENT default page reload
  IF name is empty OR email is invalid:
    SHOW an error message next to the field
  ELSE:
    CLEAR the form and SHOW a confirmation message
END ON submit

Wireframes

I'm planning two pages for now, a home page and a products page. There's a wireframe for each one below.

Landing page

The home page has the header and nav at the top, then a hero image with a headline and a "Shop Now" button. Under that is a small row of featured products that comes from the data array, and then a footer with contact info and social links.

Products page

The products page uses the same header and nav. Below that is a bar with the filter buttons and the price sort, and then the product grid that JavaScript builds from the array. Every card has an image, the name, the price, and an "Add to cart" button.
